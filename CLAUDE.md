# 콘텐츠 리라이팅 가이드라인

이 문서는 edu-guide 프로젝트의 콘텐츠 리라이팅 작업에 대한 완전한 가이드입니다.
**새로운 세션에서도 반드시 이 문서를 먼저 읽고 작업을 진행하세요.**

---

## 0. 작업 원칙 (최우선 순위)

> **꼼꼼함이 속도보다 우선입니다.**

### 핵심 원칙
- **작업량이 많더라도 시간을 줄이기 위해 임의로 간추리거나 생략하지 않습니다**
- **정해진 룰에 맞춰 꼼꼼하게 작업하는 것이 가장 1순위입니다**
- **모든 파일을 빠짐없이 확인하고 수정합니다**

### 금지 사항
- ❌ 시간 절약을 위해 일부 파일만 수정하고 나머지 생략
- ❌ "대표적인 몇 개만" 수정하고 완료 처리
- ❌ 규칙을 단순화하거나 축소 해석
- ❌ 사용자가 요청한 범위보다 좁게 작업

### 올바른 작업 방식
- ✅ 요청받은 모든 파일을 하나씩 확인하고 수정
- ✅ 가이드라인에 명시된 규칙을 그대로 적용
- ✅ 작업 완료 후 누락된 파일이 없는지 재확인
- ✅ 시간이 오래 걸리더라도 품질 우선

## 1. 리라이팅의 핵심 의도

### 절대 잊지 말아야 할 것
- **템플릿 복사 + 지역명만 변경 = 잘못된 방식 ❌**
- **같은 정보라도 표현을 다르게 = 올바른 방식 ✅**

### 목표
각 파일이 마치 **다른 사람이 쓴 블로그 글**처럼 보이게 하는 것.
- 같은 주제, 같은 정보를 다루더라도
- 문장 표현, 비유, 설명 순서, 서두/마무리가 다르게
- SEO에서 중복 콘텐츠로 인식되지 않도록

### 예시: 같은 내용의 다른 표현
```
# "고등학교 수학이 어렵다"는 내용을 다르게 표현하는 방법

버전 A: "고등학교에 올라가면 수학이 확 어려워집니다."
버전 B: "중학교 때 수학 잘했는데, 고등학교 가니 갑자기 안 되네요."
버전 C: "수학 때문에 머리 아프시죠? 고등학교 수학은 차원이 다릅니다."
버전 D: "고등학교 수학, 왜 이렇게 어려운 걸까요?"
버전 E: "많은 학생들이 고등학교에 올라와서 수학의 벽에 부딪힙니다."
```

## 2. 절대 변경 불가 요소 (고정 정보)

> ⚠️ 아래 항목들은 어떤 경우에도 변경하면 안 됩니다.

### 2-1. YAML 프론트매터의 title 필드 (페이지 제목)
**SEO 키워드가 포함된 페이지 제목은 변경 금지!**
```yaml
# 이 형식 그대로 유지 (지역명만 해당 동네로)
title: "종로구 가회동 고등 수학과외 | 경복고·동성고·중앙고 내신·수능 대비"
title: "종로구 가회동 고등 영어과외 | 경복고·동성고·중앙고 내신·수능 대비"
```
- `[구] [동] 고등 [과목]과외 | [학교들] 내신·수능 대비` 형식 유지
- 키워드 순서, 구분자(|, ·) 변경 금지

### 2-2. YAML 프론트매터의 다른 필드들
```yaml
# 모두 그대로 유지
date: 2025-01-28
categories:
  - 고등교육
regions:
  - 서울
cities:
  - 종로구
description: "..." # 기존 내용 유지
tags: [...] # 기존 태그 유지
featured_image: "..." # 기존 이미지 유지
```

### 2-3. 과외 비용 (학년별 통일 - 과목 구분 없음)

> ⚠️ 과목(수학/영어)에 따라 금액이 달라지지 않습니다. 학년별로만 구분합니다.
> ⚠️ **중요**: 물결표(~) 대신 **하이픈(-)**을 사용하세요! (취소선 렌더링 방지)

```
# 초등학생
주1회 12만원 - 22만원 / 주2회 25만원 - 38만원

# 중학생
**중학생**은 주1회 22만원 - 32만원, 주2회 29만원 - 47만원 선입니다.

# 고등학생 (고1-2)
**고1-2**는 주1회 25만원 - 36만원, 주2회 33만원 - 53만원 선입니다.

# 고3
**고3**은 주1회 28만원 - 40만원, 주2회 37만원 - 59만원이 일반적입니다.
```

### 2-4. 지역별 학교명
각 지역에 해당하는 학교는 정확히 유지:
- **종로구**: 경복고, 동성고, 중앙고
- **강남구**: 휘문고, 단대부고, 중동고, 숙명여고 등 (파일별 확인)
- **기타 지역**: 해당 파일의 기존 학교 정보 유지
- 학교 특징(시험 난이도, 출제 경향 등)의 **사실 정보**도 유지

### 2-5. CTA 숏코드
```
{{< cta-dual type="final" >}}
```
- **반드시 이것만 사용**
- 중간 CTA (`type="middle"`) 절대 사용 금지
- 위치: 본문 중간~하단에 1회만

### 2-6. 아이보리 박스 HTML 구조
```html
<div style="background-color: #FDF8F0; border-left: 3px solid #d4a574; padding: 18px; margin: 15px 0; font-size: 0.95em;">
<strong>이렇게 수업합니다!</strong><br>
[내용]
</div>
```
- HTML 태그, 스타일, "이렇게 수업합니다!" 문구는 변경 금지
- **[내용] 부분만 변경 가능** (아래 섹션 3 참고)

## 3. 변경 가능/변경해야 하는 요소

> ✅ 아래 항목들은 파일마다 다르게 작성해야 합니다.

### 3-1. 서두 (YAML 다음 첫 문장/문단)

**위치**: YAML `---` 바로 다음에 오는 첫 문단

**현재 문제**: 모든 파일이 동일함
```
"고등학교 수학, 중학교 때와 완전히 다르네요."
```

**해결**: 파일마다 다른 서두 사용
```
# 수학 과외 서두 예시 (파일마다 다르게)
- "고등학교 수학, 중학교 때와 완전히 다르네요."
- "수학 성적이 갑자기 떨어졌나요?"
- "고등학교에 올라온 후 수학이 어려워졌다는 말, 정말 많이 듣습니다."
- "내 아이 수학, 왜 이렇게 안 오르는 걸까요?"
- "수포자가 되기 전에, 지금 바로 잡아야 합니다."
- "수학만 잘해도 대학이 달라집니다."
- "고등학교 수학, 어디서부터 잡아야 할지 모르겠다면"

# 영어 과외 서두 예시 (파일마다 다르게)
- "고등학교 영어, 중학교 때와 완전히 다르네요."
- "영어 성적이 오르지 않아 고민이신가요?"
- "수능 영어 1등급, 생각보다 어렵지 않습니다."
- "영어는 꾸준함이 답입니다. 지금 시작하세요."
- "고등학교 영어, 막막하다면 함께 시작해요."
- "영어 때문에 대학 못 가면 너무 억울하지 않나요?"
```

### 3-2. 본문 내 섹션 제목 (H2, H3)

**⚠️ 주의: YAML의 title 필드가 아님!**

**위치**: 본문 내의 `##`, `###` 제목들

**현재 문제**: 모든 파일이 동일한 섹션 제목 사용
```markdown
## 고등학교 수학, 무엇이 달라질까요?
### 추상적 개념의 등장
### 수능형 문제의 난이도
```

**해결**: 같은 내용이라도 제목을 다르게 표현
```markdown
# "고등학교 수학 특징" 섹션 제목 변형
## 고등학교 수학, 무엇이 달라질까요?
## 고등 수학, 왜 이렇게 어려운 걸까요?
## 중학교와 완전히 다른 고등 수학
## 고등학교 수학이 어려운 진짜 이유
## 고등학교 올라오면 수학이 안 되는 이유

# 하위 섹션도 변형
### 추상적 개념의 등장
### 갑자기 어려워지는 개념들
### 눈에 안 보이는 수학의 시작
```

### 3-3. 본문 설명 방식 (문단 내용)

**위치**: 각 섹션의 본문 내용

**현재 문제**: 모든 파일이 동일한 설명
```
수능 수학은 사고력과 응용력이 핵심입니다. 킬러 문항(21번, 29번, 30번),
시간 압박(100분 30문항), 복합 개념 응용, 함정 선지 등이 특징입니다.
```

**해결**: 같은 정보를 다른 방식으로 설명
```
# 킬러 문항 설명 변형
버전 A: "킬러 문항(21번, 29번, 30번)은 수능에서 가장 어려운 문제입니다."
버전 B: "수능 수학에서 21번, 29번, 30번은 '킬러'라고 불립니다. 이 세 문제가 1등급과 2등급을 가릅니다."
버전 C: "많은 학생들이 킬러 문항에서 무너집니다. 21번, 29번, 30번 - 이 세 문제만 풀 수 있다면 1등급입니다."
버전 D: "수능에서 가장 무서운 건 21번, 29번, 30번입니다. 킬러라고 불리는 이 문제들이 등급을 결정합니다."
```

### 3-4. 아이보리 박스 내용

**위치**: `<div>` 안의 `<strong>이렇게 수업합니다!</strong><br>` 다음 내용

**⚠️ HTML 구조와 "이렇게 수업합니다!" 문구는 유지, 내용만 변경**

**현재 문제**: 모든 파일이 동일한 박스 내용
```
첫 수업에서 학생의 수학 실력을 정확하게 진단합니다. 개념 이해도,
계산 정확도, 문제 해결 속도를 파악하고, 부족한 부분부터 차근차근 채워갑니다.
```

**해결**: 같은 의미를 다른 표현으로
```
# 첫 번째 박스 (진단) 변형
버전 A: "첫 수업에서 학생의 수학 실력을 정확하게 진단합니다. 개념 이해도, 계산 정확도, 문제 해결 속도를 파악합니다."
버전 B: "수업 시작 전, 학생이 어디서 막히는지 정확히 파악합니다. 무작정 진도를 나가지 않습니다."
버전 C: "먼저 학생의 현재 실력을 꼼꼼히 체크합니다. 어떤 개념이 부족한지, 어디서 실수가 나는지 진단합니다."
버전 D: "진도보다 진단이 먼저입니다. 학생이 무엇을 알고 무엇을 모르는지 정확히 파악한 후 수업을 시작합니다."
```

### 3-5. 학교 설명 문장 (표현만 변경, 사실은 유지)

**위치**: 각 학교 소개 섹션

**⚠️ 학교의 특징(사실 정보)은 유지, 표현만 변경**

**현재 문제**: 모든 파일이 동일한 학교 설명
```
경복고는 종로구에 위치한 명문 고등학교입니다. 심화 개념 문제 출제가 많고
경시대회 수준 문제가 포함됩니다.
```

**해결**: 같은 사실을 다른 표현으로
```
# 경복고 설명 변형 (사실은 동일: 심화 문제, 경시 수준, 시간 부족)
버전 A: "경복고는 종로구에 위치한 명문 고등학교입니다. 심화 개념 문제 출제가 많고 경시대회 수준 문제가 포함됩니다."
버전 B: "명문 경복고는 시험 난이도가 높기로 유명합니다. 경시대회 수준의 심화 문제가 출제되어 시간이 절대적으로 부족합니다."
버전 C: "경복고 시험은 만만치 않습니다. 심화 문제에 경시 유형까지, 시간 안에 다 풀기가 어렵습니다."
버전 D: "종로구 명문 경복고, 내신 시험이 특히 어렵습니다. 경시대회에서나 볼 법한 문제가 출제됩니다."
```

### 3-6. FAQ 질문/답변 표현

**위치**: `## 자주 묻는 질문` 섹션

**⚠️ 질문의 의미와 답변의 핵심 정보는 유지, 표현만 변경**

**현재 문제**: 모든 파일이 동일한 FAQ
```
**Q. 내신과 수능 중 어떤 것을 먼저 준비해야 하나요?**

고1~2는 내신 위주로, 고3은 내신과 수능을 병행합니다.
```

**해결**: 같은 질문/답변을 다른 표현으로
```
# 질문 변형
질문 A: "내신과 수능 중 어떤 것을 먼저 준비해야 하나요?"
질문 B: "내신이 중요한가요, 수능이 중요한가요?"
질문 C: "내신과 수능, 어디에 더 집중해야 할까요?"
질문 D: "고등학교 때 내신이랑 수능 비중을 어떻게 잡아야 하나요?"

# 답변 변형 (핵심: 고1~2 내신 위주, 고3 병행)
답변 A: "고1~2는 내신 위주로, 고3은 내신과 수능을 병행합니다."
답변 B: "학년에 따라 다릅니다. 고1~2는 내신에 집중하고, 고3이 되면 수능 비중을 높여갑니다."
답변 C: "둘 다 중요하지만, 고1~2 때는 내신을 잡고 고3 때 수능에 집중하는 게 효율적입니다."
```

### 3-7. 마무리 문단

**위치**: 파일 맨 마지막 `## 마무리` 섹션

**현재 문제**: 모든 파일이 동일한 마무리
```
종로구 삼청동 고등학생 수학, 지금 시작하세요. 고등학교 수학은 개념 이해,
문제 풀이력, 시간 관리가 조화를 이루어야 합니다.
```

**해결**: 파일마다 다른 마무리
```
버전 A: "종로구 삼청동 고등학생 수학, 지금 시작하세요."
버전 B: "지금이 바로 시작할 때입니다. 삼청동 고등학생 여러분의 수학 실력 향상을 응원합니다."
버전 C: "수학은 시작이 반입니다. 오늘 상담받고, 내일부터 달라지세요."
버전 D: "더 늦기 전에 시작하세요. 삼청동에서 수학 잘하는 학생, 여러분도 될 수 있습니다."
버전 E: "고등학교 수학, 혼자 고민하지 마세요. 함께하면 달라집니다."
```

## 4. NEW 포맷 규칙

## 14. 파일 생성 후 품질 검사 (필수)

> ⚠️ **중요**: 스크립트로 파일 생성 후 반드시 아래 검사를 수행하세요!

### 14-1. CTA 숏코드 검사

**올바른 형식**: `{{< cta-dual type="final" >}}` (중괄호 2개씩)
**잘못된 형식**: `{< cta-dual type="final" >}` (중괄호 1개) ❌

```bash
# 잘못된 CTA 검사 (결과가 0이어야 정상)
grep -rEl '^\{< cta|[^{]\{< cta' content/ | wc -l

# 잘못된 파일 목록 확인
grep -rEl '^\{< cta|[^{]\{< cta' content/

# 일괄 수정 (필요시)
find content -name "*.md" -exec sed -i 's/{< cta-dual type="final" >}/{{< cta-dual type="final" >}}/g' {} \;
```

### 14-2. 아이보리 박스 HTML 검사

**올바른 형식**:
```html
<div style="background-color: #FDF8F0; border-left: 3px solid #d4a574; padding: 18px; margin: 15px 0; font-size: 0.95em;">
```

```bash
# 아이보리 박스 있는지 확인
grep -l "background-color: #FDF8F0" content/high/[구이름]-*.md | wc -l
```

### 14-3. YAML 필수 필드 검사

```bash
# title 필드 확인
grep -L "^title:" content/[폴더]/*.md

# date 필드 확인
grep -L "^date:" content/[폴더]/*.md

# featured_image 확인
grep -L "featured_image:" content/[폴더]/*.md
```

### 14-4. 파일 생성 후 체크리스트

스크립트로 파일 생성 후 반드시 확인:

```
[ ] CTA 숏코드가 {{< >}} 형식인가? (중괄호 2개씩)
[ ] 아이보리 박스 HTML이 올바른가?
[ ] YAML 필수 필드(title, date, featured_image)가 있는가?
[ ] 이미지 URL에 ?w=1200&h=630&fit=crop 파라미터가 있는가?
[ ] 비용 정보가 정확한가? (수학/영어, 중등/고등 구분)
[ ] 파일 수가 예상과 맞는가? (동 수 × 2과목 × 2)
```

### 14-5. 전체 사이트 검사 명령어

```bash
# 전체 파일 수 확인
find content -name "*.md" | wc -l

# 폴더별 파일 수
for dir in content/*/; do echo "$(basename $dir): $(find "$dir" -name "*.md" | wc -l)"; done

# 잘못된 CTA 전체 검사
grep -rEl '^\{< cta|[^{]\{< cta' content/ | wc -l

# 이미지 중복 검사 (3회 이상 사용)
grep -rh "featured_image:" content/ | sort | uniq -c | sort -rn | awk '$1 >= 3' | head -20
```

## 15. 마크다운 특수문자 주의사항 (취소선 방지)

> ⚠️ **매우 중요**: 마크다운에서 `~텍스트~`는 취소선으로 렌더링됩니다!

### 15-1. 문제 상황
```
# 이렇게 쓰면 취소선이 됨 ❌
고1~2 → "~2"가 취소선 시작으로 인식
25만원~36만원 → "~36만원"이 취소선으로 표시
개포1동~개포4동 → "~개포4동"이 취소선
1~2개월, 2~3시간 → 모두 취소선 문제 발생
```

### 15-2. 해결 방법
```
# 물결표(~) 대신 하이픈(-) 사용 ✅
고1-2
25만원 - 36만원
개포1동-개포4동
1-2개월, 2-3시간
```

### 15-3. 변환 규칙
| 잘못된 표현 ❌ | 올바른 표현 ✅ |
|--------------|--------------|
| 고1~2 | 고1-2 |
| 중1~2 | 중1-2 |
| 25만원~36만원 | 25만원 - 36만원 |
| 개포1동~개포4동 | 개포1동-개포4동 |
| 1~2개월 | 1-2개월 |
| 2~3시간 | 2-3시간 |

### 15-4. 파일 생성 후 검사
```bash
# 물결표 사용 여부 검사 (결과가 0이어야 정상)
grep -c "~" content/[폴더]/[파일명].md

# 전체 폴더 검사
grep -rl "~" content/[폴더]/ | wc -l

# 물결표 일괄 변환 (필요시)
find content -name "*.md" -exec sed -i 's/~/-/g' {} \;
```

### 15-5. 요약
- **절대 물결표(~) 사용 금지**
- 범위 표현: 하이픈(-) 사용
- 가격 범위: `25만원 - 36만원` (하이픈 양쪽 공백)
- 학년 범위: `고1-2`, `중1-2` (하이픈만)
- 동 범위: `개포1동-개포4동` (하이픈만)

## 16. 깨진 이미지(액박) 수정 가이드

> 사용자가 "이 URL 이미지 깨진 것 같아" 또는 "이 페이지 이미지가 안 떠"라고 하면 아래 절차를 따르세요.

### 16-1. 문제 확인

사용자가 알려준 페이지/URL에서 이미지 ID 추출:
```bash
# 페이지 제목으로 파일 찾기
grep -rl "페이지제목" /home/user/edu-guide/content/

# 해당 파일의 이미지 URL 확인
grep "featured_image:" /home/user/edu-guide/content/[폴더]/[파일명].md
```

### 16-2. 영향받는 파일 수 확인

```bash
# 해당 이미지 ID가 사용된 모든 파일 찾기
grep -rl "photo-[이미지ID]" /home/user/edu-guide/content/
```

### 16-3. 대체 이미지 선택

사용 횟수가 적은 이미지로 교체 권장:
```bash
# 사용 횟수가 적은 이미지 확인
grep -rh "featured_image:" /home/user/edu-guide/content/ | sort | uniq -c | sort -n | head -20
```

### 16-4. 일괄 교체

```bash
# 깨진 이미지를 새 이미지로 일괄 교체
find /home/user/edu-guide/content -name "*.md" -exec sed -i 's/photo-[깨진이미지ID]/photo-[새이미지ID]/g' {} \;

# 교체 확인
grep -rl "photo-[깨진이미지ID]" /home/user/edu-guide/content/ | wc -l  # 0이어야 함
grep -rl "photo-[새이미지ID]" /home/user/edu-guide/content/ | wc -l
```

### 16-5. 수정 이력

| 날짜 | 깨진 이미지 ID | 대체 이미지 ID | 수정 파일 수 |
|------|---------------|---------------|-------------|
| 2025-12-09 | photo-1580894906475-403935091be2 | photo-1499750310107-5fef28a66643 | 14개 |

- 현재 환경에서는 외부 네트워크 접근 제한으로 자동 검증 불가
